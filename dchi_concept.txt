자연어 처리를 이용한 디치컴퍼니 원고 유사율 검사 프로그램 컨셉

1. system_argv
[키워드] [원하는 네이버 블로그 페이지 검색할 숫자] 

2. crawl
-> 키워드 검색 
=> 네이버 블로그에서 해당 키워드에 대한 광고 블로그 글 크롤링
=> 크롤링한 데이터 .txt파일로 ./샘플/[키워드] 폴더에 저장


3. analysis
-> 문서 로딩 (키워드 + 샘플 문서 로드 & 검수 디렉터리에 있는 문서 로딩)
-> 각 샘플 문서 및 검수를 위한 문서 토큰화
	=> okt.pos를 통해 명사, 형용사, 동사 태깅(결과 형태 : list 내부의 튜플 [('명사', 'Noun'), ('형용사', 'Adjective'), ('동사', 'Verb')]
	=> 1음절 초과하는 언어만 사용(필터)
	=> 불용어에 해당하는 언어 발견 시 해당 언어 미사용(필터)
-> 토큰에 대한 TF/IDF Vectorize
-> vectorize 설정(stopwords, tokenizer 설정, ngram_range 설정(1,3), min_df 설정(2), subliner_tf 설정(true)
-> cosine 유사도 검사
	=> 각각의 문서 간 cosine 유사도 비교
	=> 딕셔너리 반환 & CSV로 저장
